What is JavaScript?

  • Cross-platform, lightweight scripting language
  • Client-side and server-side
  • Called "the language of the web" - for a reason!

---

"JavaScript can be connected to the objects of its environment to provide programmatic control over them."

Ability to interact with objects from its parent is a rare feature!

History of JS

  • 1995: At Netscape, Brendan Eich created "Mocha" in only 10 days!
  • That same year, Mocha is renamed "LiveScript" and then "JavaScript".
  • 1997: JavaScript was standardized by the ECMA as the "ECMAScript" (ES) spec.
  • 1998-2003: Various battles between browser vendors about adoption and specs! IE booo.
  • 2005: "AJAX" was coined, and the web 2.0 age begins. JavaScript renaissance!
  • 2006: jQuery 1.0 was released.
  • Netscape is now called Mozilla, makers of Firefox.
  • The name JavaScript was chosen to borrow popularity from the programming language Java. Has nothing to do with Java.
  • AJAX: Asynchronous JavaScript, allows data loading in background and out of the direct flow.
  • Standardization by ECMA allowed other browsers use the same specs.

History of JS

  • 2008ish: JavaScript isn't going away; browsers unite behind ECMA specs again.
  • 2010: Node.JS was released. Now we can use it server-side!
  • 2014-2015: Rise of "Client-side MVC frameworks"- server-side architecture (MVC) in the browser.
  • 2016: Newest ECMAScript spec (ES6) changes a lot! (But more advanced than this class.)

JavaScript Basics

Statements

Each instruction in JS is a statement, like:

console.log('Hello Class!');

After each individual statement, you must add a semicolon to end the command.


	console.log('I am glad to meet you.');
	console.log('Fuzzy Wuzzy Was A Bear');
If it's left off, it's up to the browser to decide what you meant.

Data Outputs


	// Open a popup box:
	alert('Hello World!');
	alert(2015 / 746);

	// Display a message in your console:
	console.log('Hello World!');
	console.log(14++);

	// Add something to the page:
	document.write('Hello World!');
	document.write("kittens" == "kittens");

Variables

Variables hold different types of information, like strings, numbers, and collections of data.

  • Declared (created) using the word var.
  • When first declared, it does not have a value (it is undefined).
  • Once given a starting value, it is no longer undefined. This is called initializing the variable.
  • The value of a variable can change (be updated) later (don't need to re-use var).
						
	var lsp; // declared but undefined
	var lsp = "Lumpy Space Princess"; // initialized
	lsp = "Lumpy Space Princes is my favorite!"; // updated

Naming Variables

  • Case-sensitive: ohio != Ohio
  • Must be unique
  • Must start with a letter, $, or _
    • $ is used by the jQuery library
    • _ is used by the underscore.js library
    • Best just to start with a letter
  • Can only be made of letters, numbers, $, or _
  • Best practice for multiple-word variables is to use camelcase:
    • var thisIsMyVariable;
    • Start with lowercase letter, then capitalize other words

Using a Variable

Once you have declared a variable, you can use it in your code. Just type the name of the variable.


	var numberOfKittens = 5;
	console.log(numberOfKittens);

Functions

    Define the equation, plug in some values, and get a result

Function Syntax

  • Function definition (or declaration): the entire set of keyword, name, parameters, and statements making up the function
  • Keyword function
  • Function name
  • Parameters: list of arguments the function can use, enclosed with parenthesis ()
  • Statements are enclosed with curly braces {}

	function name(argument1, argument2){
		statement;
		statement;
		statement;
	} // No ending semi-colon!

Calling Functions

Loops

Process a set of data by iterating through the set
and performing an action to each item

  • while loops
  • for loops
  • Require:
    • Set of data (a range or an array)
    • Ending condition
  • infinite loops = sadfaces
  • Forgot the ending condition! (Or it's invalid)

While loops

Repeats the same code over and over until some condition is met.

while (this thing is true) { do something }


	var numberOfTacos = 10;
	while (numberOfTacos > 0) {
		console.log('OM NOM NOM'); //Eat a taco
		numberOfTacos -= 1; //Subtract 1 and update variable
		console.log('Only ' + numberOfTacos + ' tacos left!');
	}

(numberOfTacos > 0) is the ending condition

  • (numberOfTacos) = sadface!
  • (numberOfTacos < 100) = sadface!

For loops

For loops are very similar, but you declare a counter in the statement.

for(starting condition; ending condition; counter) { do something }


	// Counts 1 to 10
	for (var i = 1; i <= 10; i++) {
		console.log(i);
	}
					

	var numberOfTacos = 10;
	for (var i = numberOfTacos; i > 0; i--) {
	  console.log('OM NOM NOM'); //Eat a taco
		numberOfTacos -= 1; //Subtract 1 and update variable
		console.log('Only ' + numberOfTacos + ' tacos left!');
	}

Iterating through Arrays

Use a for loop to easily process each item in an array.


	var catNames = ['Fluffy','Admiral Snuggles','Fizzgig','Miss Havisham'];

	for (var i = 0; i < catNames.length; i++) {
		console.log(catNames[i]) //i = index, or position in the array
	}

Putting it all together!


	var catNames = ['Fluffy','Admiral Snuggles','Fizzgig','Miss Havisham'];

	function printNames(arrayOfNames) {
		for (var i = 0; i < arrayOfNames.length; i++) {
			console.log(arrayOfNames[i])
		}
	}

	printNames(catNames);

	var dogNames = ['Sparky','Thor','Sir Droolington'];

Scope

The scope of a variable defines where it can be accessed in a program.

  • Global: can be accessed anywhere in the program
  • Local: can only be accessed within the function where it was defined
var globalVariable = "GLOBAL!";
function defineVariable() {
	var localVariable = "LOCAL!";
	console.log("1. I have access to " + globalVariable);
	console.log("2. I have access to " + localVariable);
}
defineVariable();
console.log("3. I have access to " + globalVariable);
console.log("4. Do I have access to " + localVariable + "?");

//Output
"1. I have access to GLOBAL!"
"2. I have access to LOCAL!"
"3. I have access to GLOBAL!"
Uncaught ReferenceError: localVariable is not defined(…)

Scope

Scope applies to any object defined in the global or local space.


function globalFunction() {
	console.log("I am global.");
	function localFunction() {
		console.log("I am local.");
	}
	localFunction();
}
globalFunction();
localFunction();
//Output
"I am global."
"I am local."
Uncaught ReferenceError: localFunction is not defined(…)
Note: calling globalFunction() from inside globalFunction would create an infinite loop!

Scope

Each function defines a new local space


// Global space
function globalFunction() {
	// This is a local space, my variables *can* be accessed
	// by anything inside me
	var localVariable = "LOCAL!";
	function localFunction() {
		// This is a new local space, my variables *can't* be
		// accessed by my parents
		console.log("I have access to " + localVariable);
		var uberLocalVariable = "UBER LOCAL!";
		console.log("I have access to " + uberLocalVariable);
	}
	localFunction();
	console.log("Do I have access to " + uberLocalVariable); //nope!
}
globalFunction();

Scope

But! You can grant access to a local variable using return + objects!


function globalFunction() {
	var localVariable = "LOCAL!";

	function localFunction() {
		var uberLocalVariable1 = "UBER LOCAL!";
		var uberLocalVariable2 = "UBER UBER LOCAL!";
		return {
			uberLocalVariable1: uberLocalVariable1,
			uberLocalVariable2: uberLocalVariable2
		}
	}
	console.log(localFunction().uberLocalVariable1); //YEP!
	console.log(localFunction().uberLocalVariable2); //YEP!
}
globalFunction();

Document Object Model

Representation of HTML elements in your document as Objects in the browser

Turns:

Into:

DOM

  • Referred to as a "tree" structure
  • "Traversing" the DOM
  • Individual elements are called nodes
  • Each object has
    • HTML attributes (id, class, value)
    • CSS attributes (width, height, display)
  • When accessed by JavaScript, gains JS methods
    • (.style, .on("click"), .getAttribute)

DOM Access

Your browser automatically builds a Document Object to store the DOM of a page. To change a page:

  1. Find the DOM node and store it in a variable
  2. Use methods to manipulate the node

DOM Access: By Id

You can find nodes by id using the method:


document.getElementById(id);

To find:


<img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We would use:


var imgKitten = document.getElementById('kittenPic');

DOM Access: By Tag Name

You can find certain types of HTML elements using this method:


document.getElementsByTagName(tagName);

To find:


<li>Daisy</li>
<li>Tulip</li>

We would use:


var listItems = document.getElementsByTagName('li');
for (var i = 0; i < listItems.length; i++) {
  var listItem = listItems[i];
}

DOM Access: HTML 5

In newer browsers, you can use other methods too.

Available in IE9+, FF3.6+, Chrome 17+, Safari 5+:


document.getElementsByClassName(className);

Available in IE8+, FF3.6+, Chrome 17+, Safari 5+:


document.querySelector(cssQuery);
document.querySelectorAll(cssQuery);

getElement vs. getElements

Any method that starts with "getElement" will return a single node.


document.getElementById('uniqueID'); //returns a single node

Any method that starts with "getElements" will return a array of nodes. To modify a single node, you will need to use bracket notation to select the correct one.


document.getElementsByTagName('p'); //returns multiple nodes
var specficParagraph = document.getElementsByTagName('p')[2];

DOM Nodes: Attributes

You can access and change attributes of DOM nodes using dot notation.

To change this element:


<img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We could change the src attribute this way:


var imgKitten = document.getElementById('kittenPic');
var oldSrc = imgKitten.src;
imgKitten.src = 'http://placekitten.com/100/500';

DOM Nodes: Getting and Setting Attributes

You can also use getAttribute/setAttribute:


<img id="kittenPic" src="http://placekitten.com/200/300" alt="cat"/>

We could change the src attribute this way:


var imgKitten = document.getElementById('kittenPic');
var oldSrc = imgKitten.getAttribute('src');
imgKitten.setAttribute('src', 'http://placekitten.com/100/500');

DOM Nodes: Styles

You can change page css using style

To make this CSS:


body {
  color: red;
}

Use this JavaScript:


var pageNode = document.getElementsByTagName('body')[0];
pageNode.style.color = 'red';

DOM Nodes: More Styles

Change any CSS property with a "-" to camelCase, and be sure to include a unit on any number

To make this CSS:


body {
  background-color: pink;
  padding-top: 10px;
}

Use this JavaScript:


var pageNode = document.getElementsByTagName('body')[0]
pageNode.style.backgroundColor = 'pink';
pageNode.style.paddingTop = '10px';

Isolate a node (an element on the page) and change an attribute or add a new style.

.text and .innerHTML

Each DOM node has a .text and an .innerHTML property with the HTML of all its children. You can use the property to view or change the HTML of a node.

For example, you can overwrite the entire body:


var pageNode = document.getElementsByTagName('body')[0];
pageNode.innerHTML = '<h1>Oh Noes!</h1> <p>I just changed the whole page!</p>'

Or just add some new content to the end:


pageNode.innerHTML += '...just adding this bit at the end of the page.';

DOM innerHTML continued

You can also target a particular element

To fill this HTML element:


<p id="warning"></p>

We can select the node and modify it:


var warningParagraph = document.getElementById('warning');
warningParagraph.innerHTML = 'Danger Will Robinson!';

Creating New Nodes

The document object also provides ways to create nodes from scratch:


document.createElement(tagName);
document.createTextNode(text);
document.appendChild();

Creating New Nodes: Sample Code


var pageNode = document.getElementsByTagName('body')[0];

var newImg = document.createElement('img');
newImg.src = 'http://placekitten.com/400/300';
newImg.style.border = '1px solid black';
pageNode.appendChild(newImg);

var newParagraph = document.createElement('p');
var paragraphText = document.createTextNode('Squee!');
newParagraph.appendChild(paragraphText);
pageNode.appendChild(newParagraph);
